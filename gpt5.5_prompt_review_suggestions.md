# GPT-5.5 提示词审查与改进建议

审查对象：`backend/app/ai/prompts.py`

本文档汇总了两轮审查意见：

1. 对现有提示词体系的整体评估、置信度与激进度判断。
2. 更深入的新视角设计与老视角改造建议。

---

## 一、现有提示词体系总体判断

### 1.1 总体评分

- **置信度：78/100**
  - 提示词架构完整，角色拆分清晰。
  - 能形成“观点生成 → 多视角交叉 → 操作预案”的闭环。
  - 但部分视角存在叙事化、拟人化、置信度未校准的问题。

- **激进度：72/100**
  - 整体偏实战、偏交易动作。
  - 尤其 `ANTI_QUANT_SYSTEM` 和 `TRADER_SYSTEM` 会主动生成买卖、加减仓、止盈止损方案。
  - 适合主动交易辅助，不适合作为保守投顾或自动交易决策源。

- **可用性评分：82/100**
  - 适合辅助生成技术面交易计划。
  - 不建议直接自动化执行。
  - 需要配合结构化校验、风险约束、审查器和置信度校准机制。

---

## 二、现有体系的主要优点

### 2.1 牛熊辩论结构清晰

`BULL_SYSTEM` 和 `BEAR_SYSTEM` 强制从相反方向寻找证据，有助于避免单边叙事。

优点：

- 牛派专门寻找看多证据。
- 熊派专门寻找看空或风险证据。
- 双方都必须承认反面事实。
- 输出结构紧凑，适合后续裁判 agent 汇总。

### 2.2 裁判 Agent 设计较稳

`JUDGE_SYSTEM` 要求区分：

- 双方共识；
- 双方分歧；
- 谁的证据更硬；
- verdict；
- confidence；
- scenarios；
- 操作预案；
- 风险提示。

这比单次分析更稳，能降低单模型直接下结论的偏差。

### 2.3 反量化链路有特色

`QUANT_SIMULATOR_SYSTEM` 与 `ANTI_QUANT_SYSTEM` 形成两步链路：

```text
量化因子暴露 → 模拟规则型资金行为 → 寻找拥挤交易/止损挤兑机会
```

这个视角有一定差异化价值，尤其适合短线交易者识别：

- 动量追逐；
- 均值回归；
- 放量异常；
- 拥挤交易；
- 止损挤兑；
- 错杀反抽。

### 2.4 Trader Agent 有明确安全边界

`TRADER_SYSTEM` 明确声明：

> 不做新的分析，只做排序、去重、仓位化、个性化。

这是很关键的设计。

它能防止 Trader Agent 在最后一步重新发明一套技术分析逻辑，从而造成多视角分析与最终交易建议脱节。

## 三、现有体系的主要风险

### 3.1 反量化视角过度拟人化

当前反量化链路容易把“因子暴露下可能触发的规则型交易”写成“量化机构正在想什么”。

潜在问题：

- 容易生成看似专业但不可验证的机构行为叙事。
- 容易假设市场中存在统一的“量化机构意图”。
- 容易让用户误以为模型掌握了真实资金流。

建议改造方向：

> 不要强调“机构怎么想”，改为强调“当前因子组合是否容易触发同向拥挤”。

### 3.2 Trader Agent 偏主动交易

当前 `TRADER_SYSTEM` 中存在这样的倾向：

- 无持仓 + 多数看多：给买入/回踩买入方案。
- 无持仓 + 多数看空：仍要求给下方支撑位的买入场景。

这会使系统倾向于“总要给一个交易动作”，而不是充分允许“不交易”。

风险：

- 弱化观望。
- 增加过度交易倾向。
- 在不确定性高时仍给用户制造行动冲动。

建议增加硬约束：

> 当三视角多数为 `bearish` 或 `caution`，且用户无持仓时，允许 `overall_stance="wait"`，且可以不提供买入动作。

### 3.3 JSON Schema 不是严格 JSON

多处提示词中的 schema 使用了：

- 注释；
- 省略号；
- 类型说明；
- 自然语言约束。

这对大模型可读，但对程序解析不稳定。

建议：

- 用 Pydantic 或 JSON Schema 在代码层强校验。
- 提示词里保留人类可读 schema。
- 下游对字段缺失、类型错误、非法枚举值做自动修复或重试。

### 3.4 Confidence 未校准

所有 agent 都输出 `confidence`，但没有定义不同数值区间的含义。

风险：

- 模型可能天然偏高置信度。
- 不同 agent 的 confidence 不可比。
- Trader Agent 可能把虚高置信度用于仓位化。

建议增加统一校准规则：

```text
0.00-0.30：数据不足或信号混乱，只能观察。
0.31-0.50：弱倾向，不能作为主要交易依据。
0.51-0.70：中等倾向，有部分指标共振。
0.71-0.85：较强倾向，多周期或多视角共振。
0.86-1.00：极强倾向，仅在数据完整、趋势/量能/风险均确认时使用。
```

### 3.5 价位来源约束仍不够精确

提示词要求价位和量能条件必须来自输入数据，但没有定义“可导出”的具体规则。

风险：

- 模型可能自行推导支撑位、压力位。
- 模型可能用模糊表达包装未经输入验证的价位。
- 场景条件难以程序化审查。

建议：

- 明确允许使用的价位来源：`close`、`MA5/10/20/60`、`BOLL 上下轨`、`ATR stop`、`recent_high/low`、已有 scenarios。
- 禁止使用输入中没有的“前高”“缺口”“箱体上沿”等概念，除非这些字段已明确提供。

## 四、分模块可信度判断

### 4.1 综合牛熊裁判

- **可信度：75/100**
- 优点：结构清晰，能减少单边偏见。
- 风险：牛熊双方都被要求尽力论证，弱证据可能被包装得过强。

### 4.2 反量化模块

- **可信度：65/100**
- 优点：能补充拥挤交易、止损挤兑、反向交易思路。
- 风险：机构行为画像证据链较弱，拟人化较重。
- 建议：降低其在最终决策中的权重，改造成拥挤度/脆弱性检测器。

### 4.3 反身性模块

- **可信度：72/100**
- 优点：能描述市场心理、反馈循环、预期自我强化或反转。
- 风险：容易变成故事化解释。
- 建议：强制区分“叙事假设”和“量价证据”。

### 4.4 Trader Agent

- **可信度：80/100**
- 优点：只做整合、排序、去重、仓位化，边界较清晰。
- 风险：偏主动交易，可能弱化“空仓观望”。
- 建议：加入交易剧本结构和不交易规则。

### 4.5 整体激进度

- **整体建议激进度：70/100**
- 系统更像“条件触发型交易辅助系统”，而不是保守分析系统。
- 适合短/中线主动交易者。
- 不适合低频长期投资者或风险承受能力较低的用户。

## 五、推荐新增视角：流动性 / 筹码压力视角

### 5.1 为什么需要这个视角

当前体系已有：

- 牛熊技术面；
- 量化/反量化；
- 反身性；
- Trader 执行。

但缺一个非常实战的视角：

> 谁可能被迫交易？

很多短线波动不是因为“看多/看空逻辑”本身，而是因为：

- 止损盘被打出；
- 解套盘集中卖出；
- 追涨盘被套；
- 突破失败后多头被迫撤退；
- 跌破关键位后趋势资金被迫减仓；
- 缩量反弹时卖盘暂时枯竭。

这类信息对 Trader Agent 很有价值，因为最终交易不是只判断方向，还要判断：

- 哪里容易成交；
- 哪里容易被骗；
- 哪里风险收益比最好；
- 哪里是被迫买卖最集中的区域。

### 5.2 建议新增系统提示词

建议新增：`LIQUIDITY_PRESSURE_SYSTEM`

核心任务：

```text
基于输入中的价格、均线、BOLL、ATR、换手率、量比、近期高低点、已有 scenarios，
判断当前股票在未来 1-5 个交易日的流动性压力、筹码压力和被迫交易区域。
```

它应回答：

- 哪些价位附近可能触发止损、止盈、解套盘、追涨盘？
- 当前上涨/下跌是主动买入推动，还是卖盘枯竭后的被动反弹？
- 如果明天高开/低开，哪类资金最可能被迫行动？
- 当前最危险的不是方向，而是哪种交易者会被挤出去？

### 5.3 建议输出 Schema

```json
{
  "liquidity_verdict": "easy_up | hard_up | easy_down | hard_down | balanced",
  "confidence": 0.0,
  "summary": "一句话总结当前最核心的流动性压力",
  "pressure_map": [
    {
      "zone": "15.20-15.50",
      "type": "stop_loss_cluster | trapped_supply | breakout_chase | profit_taking | liquidity_void",
      "who_is_forced": "短线追涨盘 / 前高套牢盘 / 趋势止损盘 / 浮盈盘",
      "expected_reaction": "放量突破则追涨，冲高缩量则回落",
      "evidence": ["close=15.30 接近 MA20=15.20", "turnover=8.5%"],
      "confidence": 0.0
    }
  ],
  "best_trade_location": "最有交易性价比的位置",
  "worst_trade_location": "最容易被诱多/诱空的位置",
  "scenarios": [
    {
      "trigger": "收盘跌破 MA20(15.20) 且量比>1.5",
      "action": "回避追高，等待止损盘释放后再评估",
      "direction": "bearish",
      "probability": 0.0,
      "conditions": [
        {"kind": "price", "op": "<=", "value": 15.20, "target": "close"},
        {"kind": "volume_ratio", "op": ">=", "value": 1.5}
      ]
    }
  ]
}
```

### 5.4 该视角的价值评分

- **新增流动性视角收益：85/100**
- 对短线交易尤其有价值。
- 能补上现有体系中“交易位置质量”和“被迫交易区域”的盲区。

## 六、老视角改造建议

## 6.1 牛熊辩论：从“律师式论证”改成“证据权重制”

当前问题：

`BULL_SYSTEM` 和 `BEAR_SYSTEM` 都被要求“尽最大可能找证据”。这会自然产生律师式论证：

- 强证据会被强调；
- 弱证据也可能被包装；
- 裁判需要额外判断哪些证据真的硬。

建议给每条论据增加证据强度字段：

```json
{
  "claim": "站上 MA20，短线结构改善",
  "evidence": "close=15.80 > MA20=15.30",
  "strength": "strong | medium | weak",
  "failure_mode": "若明日跌回 MA20 下方则该论据失效"
}
```

裁判可以按证据权重汇总：

- 价格结构：30%
- 量能确认：25%
- 周线趋势：20%
- 大盘背景：15%
- 风险指标：10%

收益：

- 减少“牛熊都说得很有道理”的问题。
- 让裁判判断更可解释。
- 方便后续程序化审查 evidence 是否真的来自输入。

---

## 6.2 反量化：从“机构意图”改成“拥挤度/脆弱性检测”

当前问题：

反量化模块容易拟人化机构行为。

建议从：

```text
量化机构会怎么操作？
```

改成：

```text
当前因子暴露是否容易触发规则型资金的同向拥挤？
```

建议输出：

```json
{
  "crowding_score": 0,
  "crowded_side": "long | short | none",
  "fragility": 0,
  "squeeze_risk": "upside_squeeze | downside_flush | low",
  "anti_trade": "不要追随拥挤方向，等待拥挤失败信号",
  "evidence": [
    "20d momentum 为正且 close 接近 BOLL 上轨",
    "turnover z-score 异常升高"
  ]
}
```

收益：

- 更稳健。
- 更少幻觉。
- 更像风险检测器，而不是机构心理模拟器。

建议：

> 保留反量化，但降级为“拥挤度/脆弱性检测器”。

---

## 6.3 反身性：增加“叙事验证”

当前问题：

反身性视角很容易讲故事，把市场心理解释写成事实。

建议强制区分三层：

```json
{
  "narrative": "市场正在相信什么",
  "behavioral_evidence": "哪些数据证明参与者真的在行动",
  "price_confirmation": "价格是否已经验证这个叙事",
  "narrative_risk": "什么数据会证明这个故事是假的"
}
```

建议增加硬约束：

```text
如果没有量价证据，不允许把叙事写成结论，只能写成假设。
```

收益：

- 降低故事化解释风险。
- 让反身性更像可验证假设，而不是事后解释。

---

## 6.4 Trader Agent：从“动作清单”升级为“交易剧本”

当前 Trader Agent 已经能输出 actions，但动作列表仍可能让用户过度关注“买/卖点”。

建议升级成三段式交易剧本：

1. 现在不做什么；
2. 什么发生才做；
3. 做错了怎么退出。

建议输出：

```json
{
  "primary_plan": {
    "mode": "wait_pullback",
    "entry": "回踩 MA10 附近缩量企稳",
    "invalid": "收盘跌破 MA20 且量比>1.5",
    "size": "20%",
    "exit_if_wrong": "跌破触发位后不补仓，直接撤"
  },
  "alternative_plan": {
    "mode": "breakout_follow",
    "entry": "放量突破前高后轻仓跟随",
    "invalid": "突破后次日跌回突破位",
    "size": "10-20%",
    "exit_if_wrong": "跌回突破位减仓或退出"
  },
  "do_nothing_rule": "未触发价位前不追高"
}
```

收益：

- 更像真实交易系统。
- 更强调纪律。
- 能减少用户冲动操作。

## 七、建议新增 Skeptic / Red Team 审查器

### 7.1 为什么需要 Skeptic Agent

当前体系已经有多个分析视角，但缺少一个专门审查 AI 自己的角色。

这个角色不再分析股票，而是审查：

- 哪些结论证据不足；
- 哪些价位不是输入直接提供或明确可导出；
- 哪些建议过度交易；
- 哪些 confidence 明显虚高；
- 哪个 action 应该删除；
- 用户如果照做，最大可能亏在哪里。

### 7.2 建议新增系统提示词

建议新增：`SKEPTIC_SYSTEM`

核心任务：

```text
你不是股票分析师，而是 AI 交易建议审查官。
你的任务不是生成新观点，而是审查前面所有报告和 Trader 动作是否证据充分、风险匹配、没有过度交易。
```

### 7.3 建议输出 Schema

```json
{
  "final_confidence_adjustment": -0.15,
  "overtrading_risk": 0,
  "reject_actions": [
    {
      "action_ref": 2,
      "reason": "触发价位不是输入数据直接导出，且与主 verdict 冲突"
    }
  ],
  "downgrade_actions": [
    {
      "action_ref": 1,
      "from_size": "30%",
      "to_size": "10-15%",
      "reason": "多视角未共振，且量能确认不足"
    }
  ],
  "missing_data": ["日均成交额", "近20日高低点", "量比"],
  "unsupported_claims": [
    "报告声称存在前高压力，但输入未提供 recent_high"
  ],
  "final_warning": "当前更适合等待确认，不适合主动加仓"
}
```

### 7.4 Skeptic Agent 的价值评分

- **新增 Skeptic 审查器收益：90/100**
- 这是最值得新增的安全层。
- 它能降低模型幻觉、虚高置信度和过度交易倾向。

## 八、推荐最终架构

### 8.1 当前架构

```text
牛派 + 熊派 → 裁判
量化模拟 → 反量化
反身性
三者 → Trader
```

### 8.2 建议升级架构

```text
牛派 + 熊派 → 裁判
拥挤度/反量化
反身性
流动性/筹码压力
        ↓
Trader 生成交易剧本
        ↓
Skeptic 审查并下调置信度
        ↓
最终输出
```

### 8.3 建议最终权重

默认权重：

- 综合牛熊裁判：30%
- 流动性/筹码压力：25%
- Trader 交易位置质量：20%
- 反量化拥挤度：15%
- 反身性：10%

短线交易权重：

- 流动性/筹码压力：35%
- 综合牛熊裁判：25%
- Trader 交易位置质量：20%
- 反量化拥挤度：15%
- 反身性：5%

中线持仓权重：

- 综合牛熊裁判：40%
- 周线/趋势结构：25%
- 流动性/筹码压力：15%
- 反量化拥挤度：10%
- 反身性：10%

## 九、最高优先级改造清单

### P0：必须优先做

1. 新增 `SKEPTIC_SYSTEM`
   - 审查所有 AI 输出。
   - 降低虚高置信度。
   - 删除或降级过度交易动作。

2. 给所有 agent 增加统一 confidence 校准规则。

3. Trader Agent 增加“不交易规则”。
   - 允许在信号混乱或多数 caution/bearish 时明确 wait。

### P1：强烈建议做

1. 新增 `LIQUIDITY_PRESSURE_SYSTEM`
   - 识别筹码压力、止损盘、解套盘、诱多诱空区域。

2. 将反量化改造成拥挤度/脆弱性检测器。

3. 牛熊论据增加 `strength` 和 `failure_mode`。

### P2：后续优化

1. 反身性增加叙事验证字段。
2. Trader Agent 从 actions 升级为 primary/alternative/do_nothing 交易剧本。
3. 所有 scenario 增加 `time_horizon_days`。
4. 最终输出增加 `risk_budget_pct` 或 `max_loss_pct`。

## 十、最终建议结论

最推荐先做两件事：

1. **新增 `SKEPTIC_SYSTEM`**
   - 收益最高。
   - 能显著降低幻觉、过度交易和虚高置信度。
   - 评分：**90/100**。

2. **新增 `LIQUIDITY_PRESSURE_SYSTEM`**
